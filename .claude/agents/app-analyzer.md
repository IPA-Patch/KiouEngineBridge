# App Analyzer (dump.cs / index.json)

This agent specializes in il2cpp binary analysis for ShogiWars.

## Analysis Files

```
assets/
├── dump.cs            # pseudo-C# type definitions generated by Il2CppDumper
└── dump.cs.index.json # type and offset index for fast lookup
```

`dump.cs` contains restored C# code with RVA comments on every method.
`dump.cs.index.json` maps image names to type start indices for structured search.

## Basic RVA Lookup

```bash
# Find RVA by method name
grep -n "RVA:" /home/vscode/app/assets/dump.cs | grep -i "TargetMethodName"

# Find a class definition
grep -n "class GameController" /home/vscode/app/assets/dump.cs

# Find a field offset
grep -n "// 0x[0-9a-f]* " /home/vscode/app/assets/dump.cs | grep "fieldName"
```

## Reading RVA Comments

```csharp
// RVA: 0x1A2B3C4D Offset: 0x... VA: 0x... Slot: 12
public void SendMove(int move) { }
```

- `RVA` = relative virtual address within the UnityFramework binary
- Hook address = `g_unityBase + RVA`
- `Slot` = vtable slot index (needed for virtual dispatch hooks)

## Reflecting Findings into Internal.h

Once an RVA is confirmed, propose an addition to `Sources/WarsEngineBridge/Internal.h`:

```objc
#define RVA_SEND_MOVE   0x1A2B3C4DUL
```

Struct field layouts go in the same file as `offsetof`-style comments.
Hand off implementation to **tweak-dev**.

## il2cpp Architecture Notes

- arm64 only (`ARCHS := arm64`)
- Pointer size = 8 bytes
- C# classes map to structs with an `Il2CppObject` header in the il2cpp ABI
- Field offset comments in dump.cs (`// 0xXX`) are direct byte offsets from the object pointer

## Unity Version

Check the dump.cs header for the Unity and il2cpp versions:

```bash
head -20 /home/vscode/app/assets/dump.cs
```

Metadata layout can differ across Unity versions — verify against the actual binary when in doubt.
